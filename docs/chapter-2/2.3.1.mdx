# 2.3.1  Strings como Objetos de Dados

Em JavaScript, strings (delimitadas por aspas duplas `"` ou aspas simples `'`) permitem que programadores tratem texto como dados literais ao invés de código a ser avaliado. Por exemplo, `"z"` denota uma string de caractere, enquanto `z` sem aspas refere-se ao valor de uma variável.

<CodePlayground
  session="2.3.1"
  code={`const a = 1;
const b = 2;

list(a, b);          // usa os valores das variáveis

list("a", "b");      // usa as strings literais

list("a", b);        // mistura string e valor`}
  height={250}
/>

Podemos formar listas compostas de strings e números:

<CodePlayground
  session="2.3.1"
  code={`const one_two_three = list(1, 2, 3);

const one_two = list(1, 2);

const list_of_lists = list(list("george"),
                           list("x", "y"),
                           one_two);

head(tail(head(tail(tail(list_of_lists)))));`}
  height={250}
/>

## A Função member

Para implementar a manipulação de dados simbólicos, precisamos de uma função `member` que verifica se um item está presente em uma lista:

<CodePlayground
  session="2.3.1"
  code={`function member(item, x) {
    return is_null(x)
           ? null
           : item === head(x)
           ? x
           : member(item, tail(x));
}

member("apple", list("pear", "banana", "prune"));

member("apple", list("x", "y", "apple", "pear"));`}
  height={300}
/>

A função `member` retorna a sublista começando com a primeira ocorrência do item, ou `null` se o item não está na lista.

## Exercício 2.53

O que seria impresso pelo interpretador em resposta à avaliação de cada uma das seguintes expressões?

<CodePlayground
  session="2.3.1"
  code={`list("a", "b", "c");

list(list("george"));

tail(list(list("x1", "x2"), list("y1", "y2")));

tail(head(list(list("x1", "x2"), list("y1", "y2"))));

member("red", list(list("red", "shoes"), list("blue", "socks")));

member("red", list("red", "shoes", "blue", "socks"));`}
  hiddenCode={`function member(item, x) {
    return is_null(x)
           ? null
           : item === head(x)
           ? x
           : member(item, tail(x));
}`}
  height={350}
/>

<Solution>

Na notação de caixas desta edição (execute o bloco acima e confira):

| Expressão | Valor |
|---|---|
| `list("a", "b", "c")` | `["a", ["b", ["c", null]]]` |
| `list(list("george"))` | `[["george", null], null]` |
| `tail(list(list("x1","x2"), list("y1","y2")))` | `[["y1", ["y2", null]], null]` |
| `tail(head(list(list("x1","x2"), list("y1","y2"))))` | `["x2", null]` |
| `member("red", list(list("red","shoes"), list("blue","socks")))` | `null` |
| `member("red", list("red","shoes","blue","socks"))` | `["red", ["shoes", ["blue", ["socks", null]]]]` |

O quinto caso pega muita gente: as *cabeças* daquela lista são **listas** (`list("red","shoes")` etc.), nunca a string `"red"` — o `member` compara com `===` e não entra dentro delas.

</Solution>

## Exercício 2.54

Duas listas são consideradas `equal` se contêm elementos iguais organizados na mesma ordem. Implemente `equal` como uma função.

<Solution>

**Ideia.** Igualdade estrutural: ou os dois são pares com cabeças iguais e caudas iguais, ou são primitivos idênticos.

<CodePlayground
  code={`function equal(a, b) {
    return is_pair(a) && is_pair(b)
           ? equal(head(a), head(b)) && equal(tail(a), tail(b))
           : a === b;
}

display(equal(list("a", "b", list("c")),
              list("a", "b", list("c"))));
display(equal(list("a", "b"), list("a", list("b"))));
equal(null, null);`}
  height={340}
  showLineNumbers={false}
/>

O caso base cobre strings, números e o `null` do fim das listas de uma vez — `===` faz o trabalho quando não há mais estrutura para descer.

</Solution>

## Exercício 2.55

Explique o que acontece quando você avalia uma string com a função `display` e por que isso difere da avaliação normal.

<Solution>

Avaliar uma string sozinha — `"abracadabra";` — faz o interpretador **ecoar o valor** na sua representação de programa: entre aspas, `"abracadabra"`. Já `display("abracadabra")` **imprime o conteúdo** da string, sem aspas, e devolve a string (que o REPL então ecoa com aspas).

A diferença é entre o *valor* e a *notação que o denota*: o eco do interpretador responde "que expressão produziria este valor" (por isso as aspas, como `stringify` faz); `display` responde "que texto este valor contém". Para números a distinção é invisível — para strings, as aspas são exatamente a fronteira entre dados e código.

</Solution>
